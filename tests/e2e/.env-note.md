# e2e env note (T034)

The Playwright check-in spec drives the real Vite dev server against the running
Supabase local stack. The dev server needs two browser-exposed env vars:

- `VITE_SUPABASE_URL` — the local API URL (e.g. `http://127.0.0.1:54321`)
- `VITE_SUPABASE_ANON_KEY` — the local anon/publishable key

## How they're provided

`playwright.config.ts` resolves these at config-load time by running
`supabase status -o env` (via `execSync`) and injects them into the `webServer`
env. This keeps the e2e self-contained: as long as the local stack is up, no
manual `.env` editing is needed.

```
supabase start          # ensure the stack is running
supabase db reset       # apply migrations + seed (the-heron, demo-heron-code-v1)
npx playwright test tests/e2e/checkin.spec.ts --project=chromium
```

If `supabase` is not on PATH (CI without the CLI), the config falls back to the
documented local default URL and an empty anon key — in that case set the two
`VITE_*` vars yourself (or rely on `app/.env.local`, which Vite reads).

## Network profile

The spec runs the `chromium` project by default for the timed assertions. A
`slow3g` project also exists (see `playwright.config.ts`) that applies a 3G-class
profile via CDP `Network.emulateNetworkConditions` for the SC-002/SC-008 budget
read; run it explicitly with `--project=slow3g` when measuring the throttled path.
